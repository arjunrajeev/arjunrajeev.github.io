THE TEST ARCHITECT

The key challenge comes to designing an automation framework. The myth of an organization-wide framework no longer stands good. The framework has to suite the application, the desired tool sets used for automation, as well as the end user group. Using advanced concepts like BDD, Page Objects, Command patterns, and Assertions add heavily to the framework as well. The test automation framework is ultimately a product that helps you tackle day to day tasks, an escapede from complexities, and helps you test better.

THE TEST DESIGNER

The important consideration while writing tests is, what we have applied from our learning and how robust the tests are. I can ywrite a hundred tests that can yield very few defects or write very few tests that can yield many defects. There are different techniques that can help in writing effective tests; Orthogonal Arrays or Dependency Structure Matrix are just to name a few. Authoring effective tests is essentially a skill that is hard to master.
